# Docker Setup Documentation

This document explains how to set up and run the Vive Real Estate application using Docker containers.

## Prerequisites

- Docker installed on your system
- Docker Compose installed on your system
- Git (for cloning the repository)

## Project Structure

```
viverealestate/
├── backend/
│   ├── Dockerfile
│   └── ...
├── frontend/
│   ├── Dockerfile
│   ├── nginx.conf
│   └── ...
└── docker-compose.yml
```

## Container Architecture

The application consists of three main containers:

1. **Frontend Container**
   - Built with Next.js
   - Served by Nginx
   - Port: 3000
   - Handles static file serving and API proxying

2. **Backend Container**
   - Built with NestJS
   - Port: 3001
   - Handles API requests and business logic
   - Connects to the database

3. **Database Container**
   - PostgreSQL 14
   - Port: 5432
   - Stores application data
   - Uses persistent volume for data storage

## Configuration Files

### 1. Backend Dockerfile

```dockerfile
# Build stage
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Production stage
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install --only=production
COPY --from=builder /app/dist ./dist
COPY .env .env
EXPOSE 3001
CMD ["npm", "run", "start:prod"]
```

### 2. Frontend Dockerfile

```dockerfile
# Build stage
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Production stage
FROM nginx:alpine
COPY --from=builder /app/.next /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 3000
CMD ["nginx", "-g", "daemon off;"]
```

### 3. Nginx Configuration

```nginx
server {
    listen 3000;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 10240;
    gzip_proxied expired no-cache no-store private auth;
    gzip_types text/plain text/css text/xml text/javascript application/x-javascript application/xml application/javascript;
    gzip_disable "MSIE [1-6]\.";

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Cache static assets
    location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
        expires 7d;
        add_header Cache-Control "public, no-transform";
    }

    # Proxy API requests to backend
    location /api {
        proxy_pass http://backend:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

### 4. Docker Compose Configuration

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    ports:
      - "3001:3001"
    environment:
      - DATABASE_URL=postgresql://postgres:postgres@db:5432/vive_realestate
      - JWT_SECRET=your_jwt_secret
      - JWT_EXPIRATION=24h
      - SMTP_HOST=smtp.example.com
      - SMTP_PORT=587
      - SMTP_USER=your_smtp_user
      - SMTP_PASS=your_smtp_password
    depends_on:
      - db
    volumes:
      - ./backend:/app
      - /app/node_modules
    networks:
      - vive-network

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "3000:3000"
    depends_on:
      - backend
    volumes:
      - ./frontend:/app
      - /app/node_modules
    networks:
      - vive-network

  db:
    image: postgres:14-alpine
    ports:
      - "5432:5432"
    environment:
      - POSTGRES_USER=postgres
      - POSTGRES_PASSWORD=postgres
      - POSTGRES_DB=vive_realestate
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - vive-network

volumes:
  postgres_data:

networks:
  vive-network:
    driver: bridge
```

## Running the Application

### 1. Build and Start Containers

```bash
# Build and start all containers
docker-compose up --build

# Run in detached mode
docker-compose up -d --build
```

### 2. Stop Containers

```bash
# Stop all containers
docker-compose down

# Stop and remove volumes
docker-compose down -v
```

### 3. View Logs

```bash
# View logs for all containers
docker-compose logs

# View logs for specific service
docker-compose logs backend
docker-compose logs frontend
```

### 4. Access Services

- Frontend: http://localhost:3000
- Backend API: http://localhost:3001
- Swagger Documentation: http://localhost:3001/api
- Database: localhost:5432

## Development Workflow

### 1. Making Changes

1. The application uses volume mounts for both frontend and backend:
   ```yaml
   volumes:
     - ./backend:/app
     - /app/node_modules
   ```
   This means changes to your local files will be reflected in the containers.

2. For backend changes:
   - The NestJS application will automatically reload
   - No need to rebuild the container

3. For frontend changes:
   - Next.js will automatically rebuild
   - Nginx will serve the updated files

### 2. Database Migrations

```bash
# Run migrations
docker-compose exec backend npm run typeorm migration:run

# Revert migrations
docker-compose exec backend npm run typeorm migration:revert
```

### 3. Seeding Data

```bash
# Run seeders
docker-compose exec backend npm run seed
```

## Troubleshooting

### 1. Container Issues

```bash
# Check container status
docker-compose ps

# Restart specific service
docker-compose restart backend

# Rebuild specific service
docker-compose up -d --build backend
```

### 2. Database Issues

```bash
# Access database
docker-compose exec db psql -U postgres -d vive_realestate

# Check database logs
docker-compose logs db
```

### 3. Network Issues

```bash
# Check network
docker network ls

# Inspect network
docker network inspect vive-network
```

## Production Deployment

### 1. Environment Variables

1. Create a `.env` file for production:
   ```env
   DATABASE_URL=postgresql://postgres:secure_password@db:5432/vive_realestate
   JWT_SECRET=your_secure_jwt_secret
   JWT_EXPIRATION=24h
   SMTP_HOST=your_smtp_host
   SMTP_PORT=587
   SMTP_USER=your_smtp_user
   SMTP_PASS=your_smtp_password
   ```

2. Update docker-compose.yml with production settings:
   ```yaml
   environment:
     - NODE_ENV=production
     # ... other environment variables
   ```

### 2. Security Considerations

1. Use secrets for sensitive data:
   ```yaml
   secrets:
     db_password:
       file: ./secrets/db_password.txt
   ```

2. Enable SSL/TLS for production:
   - Configure Nginx with SSL certificates
   - Update environment variables accordingly

### 3. Scaling

1. Scale backend services:
   ```bash
   docker-compose up -d --scale backend=3
   ```

2. Use a load balancer for multiple backend instances

## Maintenance

### 1. Backup Database

```bash
# Create backup
docker-compose exec db pg_dump -U postgres vive_realestate > backup.sql

# Restore backup
docker-compose exec -T db psql -U postgres vive_realestate < backup.sql
```

### 2. Update Dependencies

```bash
# Update backend dependencies
docker-compose exec backend npm update

# Update frontend dependencies
docker-compose exec frontend npm update
```

### 3. Clean Up

```bash
# Remove unused volumes
docker volume prune

# Remove unused images
docker image prune
```
